# Maintainer Notes

## Release process

1. Ensure `main` is green (full CI matrix, benchmarks, packaging job).
2. Update the version in `pyproject.toml` and `src/agent_reliability_harness/__init__.py` (keep them in sync).
3. Move `Unreleased` entries in CHANGELOG.md under the new version with the date.
4. Rerun `python benchmarks/run.py --write` and commit the regenerated results.
5. Tag `vX.Y.Z`; CI's package job builds the sdist/wheel and verifies a
   clean-environment install.
6. **Do not publish to PyPI, create a GitHub release, or push tags without
   explicit authorization from the repository owner.**

Semantic versioning: patch = fixes with no behavior change; minor = additive
features (new rules, new optional fields); major = anything that changes
existing verdicts, scores, formats, or rule semantics — requires a migration
path documented in COMPATIBILITY.md.

## Rule ID governance

- IDs are append-only per category (`ARH-SCH-011` comes after `010`).
- Never reuse, renumber, or change the meaning of an existing ID; retired
  rules keep their ID documented as retired.
- Changing a rule's *default severity* or firing conditions is a behavior
  change: golden tests and the benchmark generator must be updated in the same
  PR, and it needs a minor (or major) version bump with a CHANGELOG entry.
- After editing `rules.py`, run `python tools/generate_rules_doc.py`.

## Benchmarks

- Cases live in `benchmarks/cases/` but are owned by
  `benchmarks/tools/generate_cases.py`; CI fails if they drift.
- `BENCHMARK-RESULTS.md` is generated by `python benchmarks/run.py --write`
  only. If thresholds fail, the product regressed — fix the code, not the
  thresholds.

## Review checklist for risky changes

- Determinism: could this read time, environment, ordering of dicts/sets, or
  filesystem state into output?
- Compatibility: does `tests/test_compat_golden.py` still pass untouched?
- Fingerprints: does the change alter `(trace_id, rule_id, category, step_id)`
  for existing findings? That silently breaks users' baselines.
- Error paths: malformed input must exit 1 with `error: ...`, never a
  traceback (`tests/test_adversarial.py`).
